# 前端修改待办清单

> 记录时间: 2026-07-07
> 来源: UI 审查和用户反馈

---

## 1. 侧边栏滚动不同步 + 个人中心不可见

**问题：** 
- 首页往下滚动到底部后，侧边栏没有同步滚下来（侧边栏固定在顶部不动）
- 很多页面需要往下滚动才能看到侧边栏底部的个人中心（"我的"入口），导致操作路径长、容易忽略

**定位：** `frontend/src/components/Layout/Sidebar.css`

**预期：** 侧边栏跟随页面滚动，和主内容同步。

---

## 2. 题目详情页高度问题

**问题：** 题目详情页的上下高度超出了屏幕高度，需要缩小高度让内容刚好填满视口。现在提交按钮在首屏之下，需要额外滚动才能看到，使用体验不合理。

**定位：** 题目详情页（`frontend/src/pages/Problems/ProblemDetail*`）

**预期：** 页面内容高度刚好填满视口，提交按钮在首屏可见，无需滚动。

---

## 3. 侧边栏游戏化分组层级

**问题：** 侧边栏"游戏化"分组下有六个按钮，但底下五个（成就、每日挑战、积分、虚拟道具、排行榜）都是归属于第一个"游戏化中心"的子页面，缺少层级关系。

**定位：** `frontend/src/components/Layout/Sidebar.tsx`

**预期：** 子页面缩进或添加层级标识，体现归属关系。

**当前结构：**
```
游戏化
├── 游戏化中心
├── 成就
├── 排行榜
├── 每日挑战
├── 积分
└── 虚拟道具
```

## 4. 游戏化子页面 UI 改造（5 页）

**问题：** 游戏化中心下属页面中除了排行榜页面设计令人满意外，其他五个页面的 UI 都比较混乱、内容嘈杂、重点不突出、不够美观。

**需要改造的页面：**
- 游戏化中心首页 (`/gamification`) — `GamificationHubPage`
- 成就页面 (`/achievements`) — `AchievementsPage`
- 每日挑战 (`/daily-challenge`) — `DailyChallengePage`
- 积分页面 (`/points`) — `PointsPage`
- 虚拟道具 (`/virtual-items`) — `VirtualItemsPage`

**参考标准：** 排行榜页面 (`/leaderboard`) 的设计风格和视觉质量。

**参考实现：** `frontend/src/pages/Gamification/LeaderboardPage.tsx`

**改造方向（参考排行榜）：**
- 清晰的卡片式布局
- 视觉层次分明，一眼可抓重点
- 干净、不嘈杂的配色和间距
- 统一的设计语言
